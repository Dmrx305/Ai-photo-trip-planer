# Roadmap

## Current Phase

The project is currently in MVP foundation mode.

Already in place:

- React + TypeScript frontend
- local Express backend
- free geodata integration
- local Ollama integration with fallback mode
- redesigned frontend presentation
- project and skill documentation

## Phase 1: Solid MVP

Focus:

- keep the current one-day planning flow reliable
- improve spot relevance
- stabilize fallback behavior
- strengthen prompt quality for Ollama

Likely tasks:

- refine Overpass scoring and diversity
- improve route readability in the UI
- tighten AI output validation
- improve error states and empty states

## Phase 2: Smarter Planning

Focus:

- better planning quality and stronger realism

Likely tasks:

- optional routing integration
- location caching
- stronger timing heuristics
- filter support for paid versus free places
- more nuanced photo-style combinations

## Phase 3: Product Expansion

Focus:

- make the project more useful beyond the first MVP

Likely tasks:

- saved plans
- export and sharing
- multi-day plans
- user preferences
- image-based style or vibe input

## Ongoing Documentation Rule

As the roadmap changes, the docs should be kept in sync, especially:

- `docs/README.md`
- `docs/architecture.md`
- `docs/api.md`
- `docs/skills/skills.md`
- `CHANGELOG.md`
